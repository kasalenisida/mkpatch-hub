# mkpatch-hub

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放和展示一系列独立的 HTML 页面。这些页面可能包含信息展示、工具页面、演示页面等，不针对任何特定域名或具体网站。仓库本身作为一个页面集合的载体，提供结构化的归档与浏览方式。

## 目录说明

```
mkpatch-hub/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
└── index.html      # 仓库主页（可选）
```

- **pages/**：每个独立 HTML 页面存放于此目录下，建议按照功能或主题子目录组织。
- **assets/**：页面所需的公共资源文件，避免页面之间重复引用。
- **index.html**：可选的入口页面，用于导航或介绍仓库内容。

## 页面归档说明

- 每个 HTML 页面应保持独立，尽量避免跨页面依赖。
- 页面文件名建议使用英文小写，多个单词使用连字符（`-`）连接。
- 页面内不要包含任何具体域名、网址、营销信息或诱导点击内容。
- 页面标题和内容应清晰反映其用途，便于浏览者理解。

## 维护说明

- 欢迎提交新页面，请确保页面内容符合本仓库的通用定位。
- 提交前请检查页面是否包含不必要的第三方资源或外部链接。
- 请保持目录结构整洁，新页面放入 `pages/` 下合适的子目录。
- 如有较大改动，建议先在 Issue 中讨论。
